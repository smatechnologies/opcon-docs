## OpCon 24.1.0

**NOTE**: Verify/Upgrade these components if applicable: **ServiceNow** Connector – 21.4 or higher, **WebServices** Connector – 21.2 or higher, **Deploy** – 22.4 or higher

2024 April

#### Installation

:white_check_mark: **OPCON-23104**: Modified SMADB_Backup script from WITH NOINIT to WITH INIT and SMADB_TLog_Backup script from WITH INIT to WITH NOINIT.

#### Server

:eight_spoked_asterisk: **OPCON-22997**: Added a new API for cloud environments that processes all messages for an on-prem relay forwarding on-prem agent messages from the cloud.

:eight_spoked_asterisk: **OPCON-23208**: Added a new Relay for OpCon Datacenter customers. Download it from the FTP site and run on an elevated command prompt. This component communicates with OpCon in the cloud and relays messages for on-prem agents, enabling cloud-based OpCon without a VPN connection.

:white_check_mark: **OPCON-22260**: Fixed an issue where the OpConMFT agent was reported as down when the job ID of a job status request was not found.

#### Solution Manager

:eight_spoked_asterisk: **OPCON-20079**: Improved formatting for z/OS Job Output menu items.

:eight_spoked_asterisk: **OPCON-23078**: Require Email for Users: Email is now required on user accounts to support password recovery. A new server option lets admins allow existing users to enter their recovery email on next login.

:eight_spoked_asterisk: **OPCON-23080**: Forgot Password Link: A "Forgot Password?" link on the login screen lets users initiate password recovery by confirming their username and email to receive a recovery code.

:eight_spoked_asterisk: **OPCON-23082**: Password Recovery Validation: Users can enter and validate the recovery code received via email and complete the password reset process.

:eight_spoked_asterisk: **OPCON-23230**: Save Daily Job Details to Master: Users can push changes from daily jobs to their associated master job from within Operations and Daily Job Details views.

:eight_spoked_asterisk: **OPCON-23234**: Tag Management View Access: Admins can now control user access to the tag management view. All existing roles receive this privilege automatically.

:eight_spoked_asterisk: **OPCON-23244**: Reset Job Types from Studio: Users can reset supported job types from the Studio designer view without navigating to the Master Job list view.

:eight_spoked_asterisk: **OPCON-23245**: Configure a Default Homepage: Users can set a custom landing page by selecting the rocket icon in the top right corner of any page. Only one homepage can be active at a time. Options include:
- Self-Service
- Vision
- Operations
- Each page in Library
- Studio

:white_check_mark: **OPCON-22318**: Fixed an issue where OpConMFT server configuration was set to false after successful registration.

:white_check_mark: **OPCON-22994**: Fixed an issue where named instance schedules could not be created when Solution Manager was in French.

:white_check_mark: **OPCON-23015**: Fixed an issue where the Save button on the daily jobs definition page was enabled without any changes.

:white_check_mark: **OPCON-23065**: Fixed an issue where users with the "Modify Jobs in Master Schedules" privilege could not save changes to master jobs.

:white_check_mark: **OPCON-23072**: Fixed an issue where PERT diagram breadcrumbs for sub-schedules were missing.

:white_check_mark: **OPCON-23106**: Fixed an issue where the "Requires XML Escape Sequences" parameter was incorrect for some agents.

:white_check_mark: **OPCON-23107**: Fixed an issue blocking tab navigation on the Access Management page when the interface was in French.

:white_check_mark: **OPCON-23108**: Fixed an issue where Users and Roles were not displayed correctly in Firefox.

:white_check_mark: **OPCON-23207**: Fixed an issue where all nodes had red borders when a schedule was first loaded. Now only nodes at position (0, 0) have red borders.

:white_check_mark: **OPCON-23247**: Fixed a runtime error when saving IBMi Spool File values in Master Jobs.

:white_check_mark: **OPCON-23248**: Fixed an issue where users could not remove frequencies from schedules.

:white_check_mark: **OPCON-23251**: Fixed an issue where the resolved date value in Self-Service user input was incorrect for the Central European Time zone.

#### REST API

:white_check_mark: **OPCON-21833**: Fixed an issue when re-establishing an authenticated connection with an OpConMFT machine.

:white_check_mark: **OPCON-22270**: Fixed a timeout when retrieving data in the Schedule and Job History report.

:white_check_mark: **OPCON-23074**: Fixed a character encoding issue that prevented using some Unicode characters when configuring jobs.

:white_check_mark: **OPCON-23349**: Fixed an issue where some role modifications did not take effect immediately for all logged-in users.

#### ImpEx2 Web Service

:white_check_mark: **OPCON-23055**: Fixed an issue during deployment that removed schedule named instances when the new schedule version had none defined.

:white_check_mark: **OPCON-23091**: Fixed an issue during deployment where a global property value was mixed up with similarly named properties.

:white_check_mark: **OPCON-23345**: Fixed an issue where schedule multi-instance values were not reset correctly during deployment when the schedule definition already existed.
