# OpCon 24.1 Release Notes

## 24.1.0

**NOTE**: For compatibility, the following packages require updating if used:

- **ServiceNow** Connector – 21.4 or higher
- **WebServices** Connector – 21.2 or higher
- **Deploy** – 22.4 or higher

*April 2024*

### Installation

:white_check_mark: **OPCON-23104: Database Backup Script Correction** Updated the `SMADB_Backup` script from `WITH NOINIT` to `WITH INIT` and the `SMADB_TLog_Backup` script from `WITH INIT` to `WITH NOINIT` to restore correct backup behavior.

### Core Engine & Infrastructure

:eight_spoked_asterisk: **OPCON-22997: Cloud Relay API** Added a new API for cloud environments that processes all messages for an on-premises relay, forwarding on-premises Agent messages from the cloud.

:eight_spoked_asterisk: **OPCON-23208: OpCon Relay for Datacenter Customers** Added a new Relay component for OpCon Datacenter customers. Download it from the FTP site and run it from an elevated command prompt. The Relay communicates with OpCon in the cloud and relays messages for on-premises Agents, enabling cloud-based OpCon without a VPN connection.

:white_check_mark: **OPCON-22260: OpConMFT Agent Status Fix** Fixed an issue where the OpConMFT Agent was reported as down when the job ID of a job status request was not found.

### Solution Manager

:eight_spoked_asterisk: **OPCON-20079: z/OS Job Output Menu Formatting** Improved formatting for z/OS Job Output menu items.

:eight_spoked_asterisk: **OPCON-23078: Required Email for User Accounts** Email is now required on user accounts to support password recovery. A new server option lets administrators allow existing users to enter their recovery email address on next login.

:eight_spoked_asterisk: **OPCON-23080: Forgot Password Link** A **Forgot Password?** link on the login screen lets users initiate password recovery by confirming their username and email address to receive a recovery code.

:eight_spoked_asterisk: **OPCON-23082: Password Recovery Validation** Users can enter and validate the recovery code received by email and complete the password reset process.

:eight_spoked_asterisk: **OPCON-23230: Save Daily Job Details to Master** Users can push changes from daily jobs to their associated master job from within the Operations and Daily Job Details views.

:eight_spoked_asterisk: **OPCON-23234: Tag Management View Access Control** Administrators can now control user access to the Tag Management view. All existing roles receive this privilege automatically upon upgrade.

:eight_spoked_asterisk: **OPCON-23244: Reset Job Types from Studio** Users can reset supported job types from the Studio designer view without going to the Master Job list view.

:eight_spoked_asterisk: **OPCON-23245: Configure a Default Homepage** Users can set a custom landing page by selecting the home button in the top-right corner of any page. Only one homepage can be active at a time. Available options include:

- Self-Service
- Vision
- Operations
- Each page in Library
- Studio

:white_check_mark: **OPCON-22318: OpConMFT Server Configuration Fix** Fixed an issue where OpConMFT server configuration was set to false after successful registration.

:white_check_mark: **OPCON-22994: Named Instance Schedule Creation in French** Fixed an issue where named instance Schedules could not be created when Solution Manager was set to French.

:white_check_mark: **OPCON-23015: Save Button State on Daily Job Definition Page** Fixed an issue where the **Save** button on the daily job definition page was enabled without any changes made.

:white_check_mark: **OPCON-23065: Master Job Save for Users with Modify Privilege** Fixed an issue where users with the **Modify Jobs in Master Schedules** privilege could not save changes to master jobs.

:white_check_mark: **OPCON-23072: PERT Diagram Breadcrumbs for Sub-Schedules** Fixed an issue where PERT diagram breadcrumbs for sub-Schedules were missing.

:white_check_mark: **OPCON-23106: Requires XML Escape Sequences Parameter Fix** Fixed an issue where the **Requires XML Escape Sequences** parameter was incorrect for some Agents.

:white_check_mark: **OPCON-23107: Tab Navigation on Access Management Page in French** Fixed an issue blocking tab navigation on the Access Management page when the interface was set to French.

:white_check_mark: **OPCON-23108: Users and Roles Display in Firefox** Fixed an issue where Users and Roles were not displayed correctly in Firefox.

:white_check_mark: **OPCON-23207: Studio Node Border Display on Initial Load** Fixed an issue where all nodes had red borders when a Schedule was first loaded in Studio. Now only nodes at position (0, 0) have red borders.

:white_check_mark: **OPCON-23247: IBMi Spool File Values in Master Jobs** Fixed a runtime error when saving IBMi Spool File values in Master Jobs.

:white_check_mark: **OPCON-23248: Remove Frequencies from Schedules** Fixed an issue where users could not remove Frequencies from Schedules.

:white_check_mark: **OPCON-23251: Self-Service Date Resolution for Central European Time** Fixed an issue where the resolved date value in Self-Service user input was incorrect for the Central European Time zone.

### REST API

:white_check_mark: **OPCON-21833: OpConMFT Authenticated Connection Re-establishment** Fixed an issue when re-establishing an authenticated connection with an OpConMFT machine.

:white_check_mark: **OPCON-22270: Schedule and Job History Report Timeout** Fixed a timeout when retrieving data in the Schedule and Job History report.

:white_check_mark: **OPCON-23074: Unicode Character Encoding in Job Configuration** Fixed a character encoding issue that prevented using some Unicode characters when configuring jobs.

:white_check_mark: **OPCON-23349: Role Modifications for Logged-In Users** Fixed an issue where some role modifications did not take effect immediately for all logged-in users.

### ImpEx2 Web Service

:white_check_mark: **OPCON-23055: Named Instance Removal During Deployment** Fixed an issue during deployment that removed Schedule named instances when the new Schedule version had none defined.

:white_check_mark: **OPCON-23091: Global Property Value Conflict During Deployment** Fixed an issue during deployment where a global property value was mixed up with similarly named properties.

:white_check_mark: **OPCON-23345: Schedule Multi-Instance Value Reset During Deployment** Fixed an issue where Schedule multi-instance values were not reset correctly during deployment when the Schedule definition already existed.
