# Operations File Transfer Messages

OpCon displays file transfer status codes alongside the completion percentage and interim transfer operations (for example, compressing or decompressing). The codes originate from the Agent participating in the file transfer job. The **Origination** column identifies the error location: **FTAgent** (Destination) or **FTServer** (Source).

## MCP SMA File Transfer Messages

The following error messages may appear in the `*SMA/FTAGENT`.

| Condition | Result |

| `*SMA/CONFIG/FILE<unique identifier>` cannot be located | Log error: Configuration file is missing. FTAGENT aborts. |
| `CFG_SMAFT_IN_USE = N` | Log error: SMAFT is not enabled for this Agent. FTAGENT aborts. |
| `CFG_SMAFT_IN_USE = O` | Log error: SMAFT is permitted for outbound file transfers only. FTAGENT aborts. |
| `CFG_SMAFT_IN_USE` is not any of the following: `N`, `O`, `I`, `B` | Log error: Invalid value for SMAFT_IN_USE flag. Flag is: *x*. FTAGENT aborts. |

## UNIX File Transfer Error Codes

File transfer errors received by STDOUT/STDERR may be viewed in the Enterprise Manager Job Output Retrieval System (JORS).

The messages in this section are output by the UNIX Agent and Server. Messages output by a non-UNIX Agent or Server differ, and you should refer to the documentation for the appropriate Agent. When the destination machine is UNIX and an error originates in the Server, the Agent prepends `FTServer:` to the message to indicate that it originated with the Server.

:::tip Example
`FTServer: Error reading Temp File [/usr/local/lsam/tmp/FT_AHhs5/daily.txt.asc.10203]`
:::

### STDOUT

| Message | Origination | Explanation |
| --- | :---: | --- |
| Original Destination File [*desf*] backed up to [*backup*] | FTAgent | Per the job's **Overwrite** option, the FTAgent backed up the Destination File before starting the file transfer. |
| Original Destination File [*file*] did not exist to backup | FTAgent | At job startup, no Destination File exists to back up. |
| Could not change default access permissions for Destination File [*desf*] | FTAgent | Following the transfer of a new Destination File, the new file's access privileges could not be changed from the system default to: full access by owner, read access by group, no access by others. If the Destination file existed before the file transfer, the file transfer does not modify its access permissions. |
| [*src_machine*]src_file --> [*dest_mach*]dest_file | FTAgent | The Destination File was created or overwritten. Also shown if the job's **Overwrite** option specified appending, but the Destination File did not exist before the file transfer. |
| [*src_machine*]src_file appended --> [*dest_machine*]dest_file | FTAgent | The Source File was appended to the Destination File. |
| `File transfer successfully completed` | FTAgent | The job ran to completion. |
| `File transfer did not complete!` | FTAgent | The job did not run to completion. The file may or may not have been transferred. Refer to the job STDERR output for details. |
